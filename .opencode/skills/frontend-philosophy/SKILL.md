# Frontend Philosophy

Apply these rules to visual and interaction work.

## Rules

1. Avoid generic, templated-looking UI.
2. Use typography with character unless the existing design system forbids it.
3. Use committed color and clear visual hierarchy.
4. Prefer purposeful motion over scattered micro-interactions.
5. Make spacing, layering, and composition feel intentional.
6. If working inside an established product or design system, preserve its visual language.

## Checklist

- Does the UI avoid generic patterns?
- Is typography intentional?
- Are color and hierarchy clear?
- Is motion purposeful?
- Does the layout feel designed?
